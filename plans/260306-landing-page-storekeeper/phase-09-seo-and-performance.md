# Phase 09 — SEO + Performance

## Context Links

- [Plan Overview](./plan.md)
- [Tech Stack Research: SEO & Core Web Vitals](../reports/researcher-3-260306-landing-page-tech-stack.md)

## Overview

- **Priority:** P2
- **Status:** pending
- **Description:** Implement comprehensive SEO meta tags, structured data (JSON-LD), Open Graph tags, sitemap generation, robots.txt, and performance optimizations targeting 95+ Lighthouse scores.

## Key Insights

- Astro ships ~50KB JS by default vs 200KB+ Next.js — performance advantage built-in
- Core Web Vitals targets: LCP < 2.5s, INP < 200ms, CLS < 0.1 (Astro baseline: 0.5s, 100ms, 0.05)
- SoftwareApplication schema enables rich snippets in Google Search
- OG tags drive social sharing; Twitter Card for X/Twitter previews
- Sitemap auto-generated by `@astrojs/sitemap` integration
- Image optimization via Astro Image (AVIF/WebP auto-conversion)

## Requirements

### Functional
- Page-specific `<title>` and `<meta description>` tags
- Open Graph meta tags (og:title, og:description, og:image, og:url, og:type)
- Twitter Card meta tags (summary_large_image)
- JSON-LD structured data: SoftwareApplication + Organization
- `sitemap.xml` auto-generated at build time
- `robots.txt` allowing all crawlers
- Canonical URL on every page
- Favicon (SVG + fallback PNG)
- Preload critical fonts and hero image
- Image optimization for all assets (AVIF/WebP with fallback)

### Non-Functional
- Lighthouse Performance: 95+
- Lighthouse Accessibility: 95+
- Lighthouse Best Practices: 95+
- Lighthouse SEO: 100
- LCP < 1.5 seconds
- CLS < 0.05
- Total page weight < 500KB (first load)
- No render-blocking JS


   ```ts
       '@type': 'Organization',
   ```

2. **Update base-layout.astro** — add structured data and preloads
   ```astro
   ---
   import { getSoftwareApplicationSchema, getOrganizationSchema } from '../lib/seo';
   // ... existing props ...
   ---
   <head>
     <!-- ... existing meta tags ... -->

     <!-- Preload critical resources -->
     <link rel="preload" as="font" type="font/woff2"
       href="/fonts/Inter-Variable.woff2" crossorigin />
     <link rel="preload" as="image" href="/images/hero-dashboard.png" />

     <!-- Favicon -->
     <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
     <link rel="icon" type="image/png" href="/favicon.png" />
     <link rel="apple-touch-icon" href="/favicon.png" />

     <!-- Structured Data -->
     <script type="application/ld+json"
       set:html={JSON.stringify(getSoftwareApplicationSchema())} />
     <script type="application/ld+json"
       set:html={JSON.stringify(getOrganizationSchema())} />

     <slot name="head" />
   </head>
   ```

3. **Create robots.txt**
   ```
   User-agent: *
   Allow: /

   Sitemap: https://storekeeper.app/sitemap-index.xml
   ```

4. **Create og-image.png**
   - Dimensions: 1200x630px
   - Content: Storekeeper logo + tagline + product screenshot
   - Keep < 200KB
   - Use Figma, Canva, or ImageMagick to generate

5. **Verify sitemap generation**
   - `@astrojs/sitemap` already added in Phase 01
   - Confirm `site` field in `astro.config.mjs` is set to production URL
   - Run `npm run build` and verify `dist/sitemap-index.xml` exists
   - Verify all pages listed: `/`, `/privacy`, `/terms`

6. **Image optimization**
   - Convert all images to AVIF/WebP using Astro's `<Image>` component:
     ```astro
     ---
     import { Image } from 'astro:assets';
     import heroImg from '../assets/images/hero-dashboard.png';
     ---
     <Image src={heroImg} alt="Dashboard" width={1200} height={800}
       format="avif" quality={80} loading="eager" />
     ```
   - For images in `public/`: manually optimize with tools like `squoosh`
   - Set explicit `width` and `height` on all `<img>` tags to prevent CLS

7. **Performance checklist**
   - [ ] No render-blocking JS (Astro handles this by default)
   - [ ] Font preloaded with `crossorigin` attribute
   - [ ] Hero image preloaded
   - [ ] All images have explicit dimensions
   - [ ] CSS is inlined for critical path (Astro handles this)
   - [ ] Alpine.js loaded with `defer` (via Astro integration)
   - [ ] No unused CSS (Tailwind JIT purges automatically)

8. **Run Lighthouse audit**
   ```bash
   npm run build
   npx serve dist -l 3000
   # Open Chrome DevTools → Lighthouse → run audit
   ```
   Target: All categories 95+

9. **Additional SEO pages** (optional)
   - Consider `src/pages/404.astro` — custom 404 page with nav back to home
   - Verify `<html lang="en">` set on all pages
   - Add `<meta name="theme-color" content="#0D9488" />` for mobile browsers

## Risk Assessment

| Risk | Impact | Mitigation |
|------|--------|------------|
| OG image not rendering on social | Medium | Test with Facebook Debugger, Twitter Card Validator |
| Hero image too large (slow LCP) | High | Compress to < 150KB, use AVIF with WebP fallback |
| Sitemap missing pages | Low | Verify after every `npm run build` |
| Structured data validation fails | Low | Test with Google Rich Results tool before deploy |

## Security Considerations

- CSP headers: consider adding via Vercel config (optional for static site)
